Receives DMX data for a single universe via sACN (Streaming ACN / E1.31).

Details

The node listens on the network for sACN packets addressed to the configured universe and emits the DMX values on its output. Depending on the selected mode it either forwards every received packet unmodified (passthrough) or merges the data of all senders using HTP (Highest Takes Precedence) or LTP (Latest Takes Precedence).

DMX values in the payload are keyed by DMX channel, starting at key 1 (not 0). Depending on the Values setting they are expressed as a percentage (0–100) or as raw DMX values (0–255).

Configuration

Universe number
The universe to observe, 1 to 63999. Default 1.
Mode passthrough | htp | ltp
Whether to emit every received packet or merged values using HTP or LTP.
Output full | changes
Emit the whole universe on every update (full) or only the changed channels (changes).
Values percent | absolute
Emit channel values as a percentage (0–100) or as raw DMX values (0–255). Default percent.
Output trigger changes | always | interval
Controls when a message is emitted, independently of the mode:
  • changes only — emit only when the values change.
  • every packet — emit on every received sACN packet, even without a change.
  • changes + cyclic — emit on change and reset a keepalive timer; if no change arrives within the interval, the current full universe is re-emitted.
In the merging modes, every packet and the cyclic re-emit send the full merged universe.
Interval number
Keepalive interval in milliseconds for the changes + cyclic trigger. Default 1000.
Blank on universe change boolean
When switching the universe via msg.universe, immediately emit a full universe of zeros for the new universe until the first real data arrives. Default false.
Interface string
IP address of the network interface used for receiving. Leave empty to listen on all interfaces.
Port number
Network port to listen on. Defaults to 5568.

Inputs

universe number
Switches the observed universe at runtime, 1 to 63999. The node stops listening on the previous universe and starts listening on the new one. Invalid values are ignored and produce a warning.

Outputs

The properties emitted depend on the selected mode.

Passthrough mode

universe number
ID of the universe the packet is addressed to.
payload object
DMX values as percentage, keyed by channel (1-based).
sequence number
Packet sequence number of the sender, 1 to 255.
source string
IP address of the sender.
priority number
Priority of the sender, 1 to 200.

Merging modes (HTP / LTP)

universe number
ID of the universe the data is addressed to.
payload object
Merged DMX values as percentage, keyed by channel (1-based).
