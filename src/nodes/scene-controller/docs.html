Records DMX scenes and plays them back later.

Details

The node stores DMX values for one or more universes as named scenes in the node's context and replays them on demand. The requested operation is selected via the action property of the incoming message:

  • save - store the payload as the given scene.
  • play - emit a previously saved scene.
  • reset - clear the given scene (or all scenes) and, if it is currently playing, emit a nulled universe.

DMX values are keyed by DMX channel, starting at key 1 (not 0). The accepted value range follows the Values setting: a percentage (0–100) or raw DMX values (0–255).

Inputs

action string
Operation to perform: save, play or reset.
scene number
The scene to save or play. Required for save and play; optional for reset (omit to reset all scenes).
universe number
The universe the payload belongs to. Required when a single universe is provided as an array or a flat value object.
payload array | object
The values to record (action save). May be an array (keys 0-511) for a single universe, an object (keys 1-512) for a single universe, or an object of universes (numeric keys) each holding an object of channel values.

Outputs

Single universe

universe number
ID of the universe being emitted.
payload object
DMX values as percentage, keyed by channel (1-based).
scene number
The scene that is played.
reset boolean
Present and true for a reset message; otherwise absent.

Multiple universes

universe undefined
Not set when multiple universes are emitted.
payload object
One object per universe, each holding DMX values as percentage keyed by channel (1-based).
scene number
The scene that is played.
reset boolean
Present and true for a reset message; otherwise absent.
