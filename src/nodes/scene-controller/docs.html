Records DMX scenes and plays them back later.

Details

The node stores DMX values for one or more universes as named scenes on disk (in the Node-RED user directory) so they survive a restart, and replays them on demand. The requested operation is selected via the action property of the incoming message:

  • save - store the payload as the given scene.
  • play - emit a previously saved scene.
  • stop - stop emitting the given scene but keep it stored. If the Send zero values on stop option is enabled and no other scene remains active, a nulled universe is emitted once.
  • reset - clear the given scene (or all scenes) and, if it is currently playing, emit a nulled universe.

DMX values are keyed by DMX channel, starting at key 1 (not 0). The accepted value range follows the Values setting: a percentage (0–100) or raw DMX values (0–255).

The Playback setting controls what happens when play is triggered while another scene is already active. In Switch mode the new scene replaces the running look. In Add mode scenes are stacked and combined per channel by HTP (highest value wins); stop or reset of a single scene then re-emits the remaining combined look. Only when the last active scene is stopped or reset does the output go dark.

Inputs

action string
Operation to perform: save, play, stop or reset.
scene number
The scene to save, play or stop. Required for save, play and stop; optional for reset (omit to reset all scenes).
universe number
The universe the payload belongs to. Required when a single universe is provided as an array or a flat value object.
payload array | object
The values to record (action save). May be an array (keys 0-511) for a single universe, an object (keys 1-512) for a single universe, or an object of universes (numeric keys) each holding an object of channel values.

Outputs

Single universe

universe number
ID of the universe being emitted.
payload object
DMX values as percentage, keyed by channel (1-based).
scene number
The scene that is played.
reset boolean
Present and true for a reset message; otherwise absent.
stopped boolean
Present and true for the nulled message emitted on stop; otherwise absent.

Multiple universes

universe undefined
Not set when multiple universes are emitted.
payload object
One object per universe, each holding DMX values as percentage keyed by channel (1-based).
scene number
The scene that is played.
reset boolean
Present and true for a reset message; otherwise absent.
stopped boolean
Present and true for the nulled message emitted on stop; otherwise absent.
